# 新手创作旅程与音频参考技术方案

本方案把“完全不懂乐理的新用户”从第一次按键，带到一首可试听、可手工修改、可由 AI 辅助扩展并可导出的作品。它不替代既有的工程文档、操作层、Piano Roll 或 AI Proposal；它只是将这些已存在的能力排成安全、可理解的任务路径。

## 用户路径

1. **音阶热身**：用户完成 Q → W → E → R → T 的小任务，直接听到从 do 到 sol 的上行变化。
2. **10 秒音乐工坊**：选择放松、轻快或有力量，生成真实 `ProjectDocument`：5 小节原创开场（120 BPM 下约 10 秒）、鼓点/低音/主旋律三轨，以及展开、变化、高潮、收束共 30 小节的路线。
3. **参考音频（可选）**：选择至多 3 个 WAV、MP3、M4A 或 AIFF 文件。长音频先在页面时间轴中选择 1–45 秒片段并试听；用户确认导入所选片段后，再为每段设置“想参考什么”和权重。
4. **创作问答**：用日常语言选择风格、能量发展、原旋律出现频率、目标时长，并可补充一句自由描述。
5. **路线图**：以开场 → 展开 → 变化 → 高潮 → 收束展示音乐结构和能量变化。
6. **逐段扩展**：可选本机引导（立即生成基础 MIDI）或 AI 候选（仅当前轨道×段落，需用户接受）。开场段不允许自动重写。
7. **手工收尾与导出**：跳转 Piano Roll 改一个音符；回到 Demo 试听、保存工程、导出 MIDI/WAV。

## 数据与隐私边界

```text
参考音频文件
  └─ Tauri 文件对话框 → 本机 WebView AudioContext 解码 → 时间轴选择 1–45 秒 → 本机试听
       └─ 用户确认导入 → 所选片段的 RMS / 过零率 / 能量曲线 → 可见、可改的文字摘要 ──→（可选）AI 提示词

工程 ProjectDocument ──→ OperationBatch ──→ 原子校验/Undo ──→ 播放/导出
AI 返回 notes ──→ Local Proposal 校验 ──→ 用户“接受” ──→ OperationBatch
```

- 原始音频字节、文件绝对路径、文件 hash 不进入 AI prompt、工程 JSON、日志或导出文件。
- 第一阶段只发送用户已确认的摘要，例如“约 12 秒、能量平衡、后段更有力量”，并在 prompt 明示不得复制参考中的旋律、歌词或音频。
- 导入文件仅为当前本机创作任务保留引用；不会复制到工程目录，也不会随 `.json` 工程保存。这样避免工程文件意外携带受版权保护的源音频。
- 创作任务当前保存在本机 localStorage，按项目 ID 关联，用于同一台 Mac 的续做。跨设备继续时，工程可正常打开，但需要重新选择参考并恢复任务选择；将任务摘要纳入工程可移植 manifest 是后续技术项。
- `contentHash` 目前是本机去重/诊断用的 FNV-1a 标识，不是安全哈希，不应用于安全校验或版权判定。

## 扩展策略

### 本机引导

本机引导为目标段落中还没有 clip 的每条轨道生成一个 `upsertClip` 操作。鼓、低音、旋律根据用户选择产生确定性基础音符。整个段落是一个 `OperationBatch`，因此：

- 原子校验失败时零写入；
- 成功后可一次 Undo；
- 不会覆写 10 秒开场或已有片段；
- 不依赖 API Key 或网络。

### AI 扩展

AI 跳转到应用内 Chat，而不是打开网页或外部网关。它预填：当前段落、用户的方向选择、经确认的参考摘要和“不修改开场”的约束。原生 Keychain、Gemini/OpenAI 选择、最小 MIDI scope、候选校验、接受/拒绝沿用既有 AI 契约。

AI 扩展候选被接受后才将该段标记完成；生成失败、Key 未配置、用户拒绝候选都不会修改工程，用户可改用本机引导或 Piano Roll。

## 资源限制与失败恢复

| 项目           | 当前限制/行为                                 |
| -------------- | --------------------------------------------- |
| 参考数量       | 最多 3 段                                     |
| 文件格式       | WAV、MP3、M4A、AIFF/AIF                       |
| 单文件限制     | 40 MB；源文件时长不限，确认导入片段上限 45 秒 |
| 解码失败       | 不改变任务，显示可行动错误并允许换文件        |
| 时间轴选择     | 拖动开始/结束手柄或填写秒数；可试听后确认导入 |
| 未同意隐私边界 | 禁止打开导入对话框                            |
| AI 不可用      | 明确显示失败；继续使用本机引导或手工编辑      |
| 已有片段       | 本机扩展不覆写；用户可改用 AI 候选或直接编辑  |

## 后续技术演进

1. 将音频特征解码从 WebView `AudioContext` 下沉为 Rust/AVFoundation worker，获得更稳定的格式支持、取消和进度。
2. 将当前双时间轴手柄升级为真实波形预览、缩放、吸附和键盘无障碍裁剪；当前版本已支持拖动/精确秒数、1–45 秒试听和仅分析所选范围。
3. 用真正的 BPM/onset/key 分析替换当前轻量 RMS/过零率估计；推断结果必须继续以“可由用户修正”为前提。
4. 设计可移植、无路径/无原始音频的 Journey manifest，并纳入项目 Schema 迁移。
5. 给本机引导加入节奏玩法的准确率/连击、无障碍替代操作与可重放学习记录。
6. 完成 AI Proposal 试听、真实 RenderPlan 一致性和最终 WAV/MIDI 端到端验收。
