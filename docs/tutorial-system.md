# 教程系统契约

教程是 MVP 的 P0 功能与发布门槛：用户不应因为不知道下一步而离开当前页面或查阅外部文档。任何功能、页面、空状态或错误状态没有对应教程，均不能标记为完成。

## 两个入口

### 1. 首次使用核心引导

首次启动展示欢迎页，可跳过但必须能从教程中心恢复。它用内置 demo 引导用户完成：

1. 认识工程与当前 section/track；
2. 播放、停止与循环；
3. 用 `Q W E R T` 试听当前调性的 do/re/mi/fa/sol；
4. 选中目标并录入一小段音符；
5. 播放结果、Undo 并恢复；
6. 在 AI 已可用时，了解 scope、发送范围和候选预览；不可用时跳过该步骤并说明本地手动创作仍可完成。

步骤必须可跳过、回退、重试和在下次启动恢复；不得强迫用户连接 AI、创建账户或完成一段音乐。

### 2. 当前页面教程（全局按钮）

固定顶栏在每个路由、空状态、错误状态和编辑状态中展示文字按钮 `当前页面教程`（可配帮助图标，但文字不可省略）。点击后打开非阻塞右侧教程栏，不离开当前工程、不改变播放或编辑状态。

教程系统以 `route + activePanel + editorMode + selection` 解析最佳教程。例如 Arrangement 的轨道/段落选择状态解释 scope，Piano Roll 解释拖动与力度，键盘录制状态解释输入映射与量化。没有精确匹配时显示该页面的基础教程和教程中心入口，绝不显示空白或报错。

## 内容标准

每个教程至少包含以下字段，内容与代码同版本维护：

| 字段 | 要求 |
|---|---|
| `id` / `contentVersion` | 稳定 ID 与可迁移版本 |
| 目标 | 用户完成后能做什么，用一句可验证的话描述 |
| 前置条件 | 所需工程、选择、权限或网络状态；无条件时明确写“无” |
| 步骤 | 1–7 个动作；每步锚定一个 UI 元素或说明无锚点原因 |
| 预期结果 | 用户可观察或听到的结果 |
| 常见错误 | 至少一个失败原因与可执行恢复动作 |
| 控制项 | 上一步、下一步、跳过、重试、退出、重新开始 |
| 无障碍文本 | 高亮元素的文字说明；不能只依赖颜色、动画或声音 |

教程内容离线内置、支持本地化，不向 AI 或第三方发送进度。进度按本地用户保存：`未开始 | 进行中 | 完成 | 已跳过`；contentVersion 变化时保留能匹配的步骤，否则从最近安全步骤恢复。

## 展示与交互规则

- 教程侧栏不遮住用户当前操作；需要强调目标时用高亮和指向层，但不得吞掉目标元素以外的点击。
- 所有控制项都可用键盘到达，焦点顺序清晰，Esc 关闭并保留进度；屏幕阅读器能读出步骤数、目标、预期结果和错误提示。
- 绑定元素使用稳定 `data-tutorial` anchor，而非文案、DOM 层级或像素坐标。锚点不可见、未挂载或窗口过小时，降级为文字说明、滚动到页面区域或“稍后重试”。
- 教程不得自行触发 AI 请求、播放、保存、删除或提交工程修改；它只能引导用户执行，并在用户操作后检测可观察的完成条件。

## 教程覆盖矩阵

| 区域/状态 | 必须覆盖 |
|---|---|
| 欢迎与 demo | 工程含义、demo、当前教程按钮、恢复/重置教程 |
| Transport / 音源 | 播放、停止、循环、音源/注册表加载失败、卡音恢复 |
| 电脑键盘 | QWERT 映射、八度、文本框不触发、试听、录制、overdub、替换范围、Undo |
| Arrangement | section、track、scope、选择错误与空状态 |
| Piano Roll / Automation | 新增、移动、删除、时值、力度、曲线与量化 |
| AI Chat | 连接状态、发送数据范围、Proposal 试听/接受/拒绝/重试、失败与离线手动替代路径 |
| 工程生命周期 | 空白/模板新建、dirty 状态、自动恢复、打开、保存、另存、损坏文档与恢复 |
| 文件与导出 | 导出全曲/片段、WAV/MIDI 差异、导出预检、进度、取消、错误与输出位置 |

## 实现边界与验收

教程是独立于工程文档的本地 `TutorialProgress` 状态；工程打开、保存、Undo、AI 操作和离线播放都不能修改其内容。App Shell 负责按钮可见性和教程中心，页面只注册上下文与 anchors。

M-1 验收：在任意已实现路由点击“当前页面教程”，都能出现带目标、步骤、预期结果、常见错误和控制项的内容；关闭/完成/跳过后主界面可继续操作，重启后进度正确恢复。每个后续里程碑验收必须再验证其新增功能的教程覆盖、无锚点降级、键盘可访问和离线可用。
