# 产品文档 — AI 音乐 IDE（MVP）

## 一句话定位

像 IDE 一样编辑音乐：以结构化工程文档为唯一真相，AI 作为 Copilot 对文档做结构化编辑，同一份文档暴露给 Chat / 编排视图 / Piano Roll 三种入口。

## MVP 范围闸门

**做**

- 器乐（无人声）符号创作
- 一句话生成完整器乐作品
- 自然语言逐步修改（"更燃一点""副歌再长一点"）
- 段落 / 轨道 / 乐器级重写
- Piano Roll + 基础 Automation 手动编辑
- 电脑键盘演奏与录制：按 `QWERT…` 映射当前调性的 do/re/mi/fa/sol，直接试听或录入选中段落
- 详尽的新手教程：首次使用的引导任务 + 所有页面随时可调出的当前页面教程
- 新手创作旅程：用 QWERT 音阶热身、10 秒音乐工坊、可选本机音频参考、日常语言问答、逐段扩展与最后导出，帮助零乐理用户独立完成第一首歌
- 完全本地化：工程存本地 `.json`
- 从空白工程或内置模板开始，自动恢复未保存工作，并导出完整成品 WAV 与可编辑 MIDI
- 用户可在 OpenAI 与 Gemini 间选择生成服务；所有配置、生成和候选接受都在同一个 macOS 应用内完成。开发验证的个人 API Key 存入 macOS Keychain；未来产品代付服务由应用原生侧透明接入，用户不需要启动网关、配置地址或在多个端之间跳转

**不做（本期砍掉的是架构假设，不是功能）**

- 人声 / 歌词 / 旋律对齐
- 真实录音 / 音频轨
- MIDI 硬件 / 专业设备输入
- 移动端
- 多人协作
- 云端存储、专业混音 / 母带

> 只要底座（文档 + 操作层）设计对，以上都是"后期加视图 / 加后端"，不需要重写核心。

## 目标用户与核心场景

| 用户 | 入口                               | 场景                                                     |
| ---- | ---------------------------------- | -------------------------------------------------------- |
| 小白 | Chat                               | "生成一段 lo-fi hiphop" → 出完整器乐 → "更燃一点" 逐步改 |
| 进阶 | 选中 section/track + Chat          | "只重写副歌的 bass"、"这段鼓换成 4-on-floor"             |
| 专业 | Piano Roll / 电脑键盘 / Automation | 演奏、录入或手改音符与曲线；AI 始终理解整个工程          |

**关键体验主张**：不存在"AI 模式"与"手动模式"的切换——三类用户操作的是**同一份文档**，AI 永远有全局上下文；也不存在“只能去外部查帮助”的时刻——每个页面都有可随时打开的教程入口。

## 产品原则

1. 符号优先，音频是"编译产物"，不是真相。
2. 每次编辑都是文档上的可逆操作（音乐版 git）。
3. 语义指令（"更燃"）只允许展开为可预览、可撤销的原语序列，绝不黑箱直改。
4. 三视图共享一份文档，作用域是唯一区分参数。
5. AI 是可选的网络能力：未配置或离线时，工程仍可打开、播放、手动编辑和保存。
6. API Key 仅可保存于 macOS Keychain；绝不读取或保存用户的 OpenAI、Gemini 或聊天产品网页密码、Cookie 或会话，也不把 Key 写入工程或日志。
7. 电脑键盘演奏、Piano Roll 与 AI 同样只通过操作层改写工程；试听不改文档，录制可完整撤销。
8. 教程是交付门槛：新页面或功能必须同时交付对应教程、常见错误与验收步骤，不能后补。
9. 用户导入的参考音频默认只在本机分析；任何 AI 请求只可使用用户确认的文字摘要，不能默认上传原始音频、文件路径或完整工程。

## MVP 成功标准

- 在 AI 发布门槛通过时，一句话能出一段 **8–16 小节、多轨、可循环播放** 的器乐；未通过时，产品明确提供等价的模板/手工创作路径而不展示 AI 生成功能。
- 至少 6 条语义指令（更燃 / 更安静 / 加长副歌 / 换鼓组 / 转调 / 重写某轨）稳定生效。
- 任意 AI 操作可一键撤销。
- Piano Roll 手改后播放正确，且 AI 后续指令仍基于改后文档。
- 全程无网络亦可完成编辑（生成步骤除外）。
- 未配置所选 AI 服务商时，应用在 Chat 页面清楚展示“可手动创作；生成需要配置 OpenAI 或 Gemini”，并提供同页的 Keychain 配置入口。
- 选中 melodic 轨道和段落后，`Q W E R T` 能按当前工程调性试听 do/re/mi/fa/sol；录制后写入可播放、可撤销的音符。
- 任意页面的固定“当前页面教程”按钮可打开对应引导；新用户能不查外部资料完成“打开 demo → 播放 → 键盘试听 → 录入 → 撤销”的核心流程。
- 用户可从空白/模板工程或 AI 片段开始，完成试听、AI/手工修改、保存恢复，并导出可播放 WAV 与可编辑 MIDI；导出前能看见不能导出的原因和修复入口。

## 非目标（明确不追求）

- 不追求母带级音质；音质到"demo 可听"即可。
- 不追求实时低延迟协作。
- 不追求覆盖所有曲风；先跑通电子 / lo-fi / 流行器乐若干种。
